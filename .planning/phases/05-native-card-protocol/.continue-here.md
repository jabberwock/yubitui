<decisions_made>

- State machine over pre-buffering: gpg prompt order is non-deterministic enough that pre-buffering caused hangs; state machine on STATUS-FD tokens is reliable
- DELETE_KEY before each keytocard session: required to clear agent's stale card reference from previous (partial) imports
- Two-field form: gpg asks for key passphrase before admin PIN; collecting both upfront lets the state machine answer in correct order
- `keyedit.save.okay` = success signal for gpg 2.4.9 (no SC_OP_SUCCESS emitted in this version)
- `cardedit.genkeys.replace_key → n` = key already on card → treat as success
</decisions_made>
