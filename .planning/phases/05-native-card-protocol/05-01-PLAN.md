Create `src/yubikey/card.rs` with the following public API. Per D-01, all card communication is raw APDUs via `pcsc` crate only. Per D-03, `pcsc = "2.8"` stays in Cargo.toml.

**Constants:**
```rust
pub const OPENPGP_AID: &[u8] = &[0xD2, 0x76, 0x00, 0x01, 0x24, 0x01];
pub const SELECT_OPENPGP: &[u8] = &[0x00, 0xA4, 0x04, 0x00, 0x06, 0xD2, 0x76, 0x00, 0x01, 0x24, 0x01];
```

**Functions to implement:**

1. `pub fn kill_scdaemon()` — runs `gpgconf --kill scdaemon`, ignores errors. Per D-04.

2. `pub fn connect_to_openpgp_card() -> Result<(pcsc::Card, Vec<u8>)>` — Per D-04/D-11: kills scdaemon, establishes Context(Scope::User), lists readers (2048-byte buffer), iterates readers trying connect(Exclusive, T0|T1), sends SELECT_OPENPGP, returns (Card, aid_data) on first success. Returns error "No YubiKey with OpenPGP application found" if none match.

3. `pub fn get_data(card: &pcsc::Card, p1: u8, p2: u8) -> Result<Vec<u8>>` — sends `[0x00, 0xCA, p1, p2, 0x00]`, checks SW==0x9000 via apdu_sw(), strips 2-byte SW from response, returns data bytes. On error: logs SW via tracing::debug!, returns apdu_error_message. Use 1024-byte buffer (safe for DO 0x6E).

4. `pub fn get_data_2byte_tag(card: &pcsc::Card, p1: u8, p2: u8) -> Result<Vec<u8>>` — same as get_data but for 2-byte tags like 0x5F50 where P1=0x5F, P2=0x50.

5. `pub fn apdu_sw(resp: &[u8]) -> u16` — moved from pin_operations.rs. Returns 0 if resp.len() < 2. Otherwise big-endian last 2 bytes.

6. `pub fn apdu_error_message(sw: u16, context: &str) -> String` — Per D-15/D-16/D-17. Maps:
   - 0x63C0..0x63CF: "Wrong PIN -- N tries remaining" (extract N from low nibble; "try" if N==1)
   - 0x6300: "Wrong PIN"
   - 0x6982: "Security condition not met -- Admin PIN required"
   - 0x6983: "Authentication method blocked"
   - 0x6A82: "Data object not found"
   - 0x6A80: "Incorrect data in command"
   - 0x6700: "Wrong length"
   - _: "Card operation failed -- try removing and reinserting your YubiKey"
   - Appends ` (context)` to all messages.

7. `pub fn serial_from_aid(aid: &[u8]) -> Option<u32>` — Per D-08. Checks len >= 14 and prefix matches OPENPGP_AID. Formats bytes 10-13 as hex string, parses as decimal u32 (BCD). Falls back to big-endian u32 if any nibble is A-F.

8. `pub fn tlv_find<'a>(data: &'a [u8], target_tag: u16) -> Option<&'a [u8]>` — Per Research Pattern 8. Walks BER-TLV: 1-byte tag if low 5 bits != 0x1F, else 2-byte tag. Length: 0x82 prefix = 2-byte length follows, 0x81 = 1-byte follows, else literal. Returns first match for target_tag.

**Module registration:** Add `pub mod card;` to src/yubikey/mod.rs (before existing modules alphabetically).

**Migration:** Change `apdu_sw` in pin_operations.rs from private `fn apdu_sw` to call `super::card::apdu_sw` instead. The private function body is removed and replaced with a call to card::apdu_sw.

Write tests in a `#[cfg(test)] mod tests` block at the bottom of card.rs covering all behaviors listed above.

**detection.rs — Replace with PC/SC reader enumeration (per D-06, D-11, D-12):**

Rewrite `detect_all_yubikey_states()`:
1. Call `card::kill_scdaemon()`
2. Establish `pcsc::Context::establish(Scope::User)`
3. `list_readers(&mut [0u8; 2048])` to get all readers
4. For each reader: `connect(reader, ShareMode::Exclusive, Protocols::T0 | Protocols::T1)`

**openpgp.rs — Replace get_openpgp_state() with native PC/SC (per D-06, D-07):**

Rewrite `get_openpgp_state()`:
1. Call `card::connect_to_openpgp_card()` to get (card, aid_data)
2. Extract version from AID bytes 6-7: `format!("{}.{}", aid_data[6], aid_data[7])`

4. Navigate TLV: `card::tlv_find(&app_data, 0x73)` for Discretionary Data Objects

   - Fingerprints: `tlv_find(disc, 0xC7)` (SIG), `tlv_find(disc, 0xC8)` (ENC), `tlv_find(disc, 0xC9)` (AUT) — each 20 bytes when present, all-zero means no key

   - Parse algorithm: first byte 0x01 = RSA (next 2 bytes = bit length big-endian), 0x12 = ECDH, 0x13 = ECDSA, 0x16 = EdDSA; for 0x12/0x13/0x16 check OID suffix for curve name

7. GET DATA 0x5F50 for URL: `card::get_data_2byte_tag(&card, 0x5F, 0x50)`
8. Build `OpenPgpState` with card_present=true, all extracted fields

On any connect failure, return `OpenPgpState { card_present: false, .. }` with all None/empty fields (same fallback as current gpg failure path).

**Algorithm attribute parsing helper** (in card.rs or openpgp.rs):
```rust
fn parse_algorithm_attributes(data: &[u8]) -> String {
    if data.is_empty() { return "Unknown".to_string(); }
    match data[0] {
        0x01 => { // RSA
            if data.len() >= 3 {
                let bits = u16::from_be_bytes([data[1], data[2]]);
                format!("RSA {}", bits)
            } else { "RSA".to_string() }
        }
        0x12 => "ECDH (Cv25519)".to_string(),  // simplified; OID check for full accuracy
        0x13 => "ECDSA (P-256)".to_string(),
        0x16 => "EdDSA (Ed25519)".to_string(),
        _ => format!("Unknown (0x{:02X})", data[0]),
    }
}
```

**Fingerprint formatting helper:**
```rust
fn format_fingerprint(fp: &[u8]) -> String {
    if fp.iter().all(|&b| b == 0) { return String::new(); }
    fp.iter().map(|b| format!("{:02X}", b)).collect::<Vec<_>>().join("")
}
```
