**touch_policy.rs — Replace with native PC/SC (per D-09, D-10):**

Add to TouchPolicy impl:
```rust
pub fn from_byte(b: u8) -> Self {
    match b {
        0x00 => TouchPolicy::Off,
        0x01 => TouchPolicy::On,
        0x02 => TouchPolicy::Fixed,
        0x03 => TouchPolicy::Cached,
        0x04 => TouchPolicy::CachedFixed,
        other => TouchPolicy::Unknown(format!("0x{:02X}", other)),
    }
}

pub fn to_byte(&self) -> u8 {
    match self {
        TouchPolicy::Off => 0x00,
        TouchPolicy::On => 0x01,
        TouchPolicy::Fixed => 0x02,
        TouchPolicy::Cached => 0x03,
        TouchPolicy::CachedFixed => 0x04,
        TouchPolicy::Unknown(_) => 0x00, // fallback to off
    }
}
```

Add new function `pub fn get_touch_policies_native(card: &pcsc::Card) -> Result<TouchPolicies>`:
- Takes `&pcsc::Card` so detection.rs can call it within the batched read path (no extra scdaemon kill).
1. For each slot DO: 0xD6 (sig), 0xD7 (enc), 0xD8 (aut), 0xD9 (att):
   - `card::get_data(card, 0x00, DO)` — response is [policy_byte, 0x20] on success
   - On SW != 0x9000: default to TouchPolicy::Off (older cards may not support UIF)

2. Return TouchPolicies { signature, encryption, authentication, attestation }

**attestation.rs — Replace with native PC/SC:**

The ykman command `ykman openpgp keys attest SLOT -` sends the GENERATE ASYMMETRIC KEY PAIR command with the attestation flag. The OpenPGP card APDU for attestation on YubiKey is:
- `00 47 81 00 0B 00 B6 00` (SIG slot attestation — INTERNAL AUTHENTICATE variant)

However, the YubiKey-specific attestation command is actually simpler. Per ykman source (openpgp.py `attest()` method), it uses:
- INS = 0xFB (YubiKey proprietary ATTEST command, NOT standard OpenPGP)
- P1 = slot CRT tag: SIG=0xB6, DEC=0xB8, AUT=0xA4
- P2 = 0x00
- The command: `[0x00, 0xFB, CRT_TAG, 0x00, 0x00]`

Map slots: "sig" -> 0xB6, "enc" -> 0xB8, "aut" -> 0xA4

Rewrite `get_attestation_cert(slot, serial)`:
1. Validate slot is in VALID_ATTEST_SLOTS
2. Call `card::connect_to_openpgp_card()`
3. Build ATTEST APDU: `[0x00, 0xFB, crt_tag, 0x00, 0x00]`
4. Transmit with a **4096-byte buffer** — DER certificate can be 1-2KB (addresses review concern: Extended Length APDUs for attestation certs)
5. SW 0x9000: response data (minus last 2 bytes) is DER-encoded certificate
6. Convert DER to PEM: base64-encode the DER bytes, wrap in `-----BEGIN CERTIFICATE-----\n...\n-----END CERTIFICATE-----`
7. SW != 0x9000: map with apdu_error_message (0x6A88 = "Key not generated on this YubiKey — attestation requires on-device key generation")

Add custom SW mapping for attestation: 0x6A88 means "referenced data not found" (key was imported, not generated on-device). This is the expected error for imported keys.

Keep `parse_attestation_result()` with `#[allow(dead_code)]` for existing unit tests.
Remove `use std::process::Command;`.

**Base64 for PEM:** Use the standard Rust approach without adding a dependency:
```rust
use base64::engine::general_purpose::STANDARD;
use base64::Engine;
```
Check if `base64` is already in Cargo.toml. If not, add it: `cargo add base64`. Alternatively, if the project doesn't use base64 elsewhere, output raw DER hex representation wrapped in a simple PEM-like format, or use the built-in approach. The simplest option that avoids a new dep: output the raw hex dump prefixed with "DER Certificate (hex):" — but PEM is standard and base64 is a tiny dep. Prefer adding base64.
