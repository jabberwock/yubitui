<task type="auto">
  <name>Task 1: Pass YubiKeyState to render_key_attributes and display touch policies</name>
  <files>src/ui/keys.rs</files>
  <action>
The root cause: `render_key_attributes(frame, area, state)` only receives `&KeyState` — it has no access to `YubiKeyState` where `touch_policies: Option<TouchPolicies>` lives.

The outer `render()` function already receives `yubikey_state: &Option<YubiKeyState>`.

**Step 1 — Update the dispatch in `render()`:**
Change:
```
KeyScreen::KeyAttributes => render_key_attributes(frame, area, state),
```
to:
```
KeyScreen::KeyAttributes => render_key_attributes(frame, area, yubikey_state, state),
```

**Step 2 — Update `render_key_attributes` signature:**
Change:
```rust
fn render_key_attributes(frame: &mut Frame, area: Rect, state: &KeyState)
```
to:
```rust
fn render_key_attributes(frame: &mut Frame, area: Rect, yubikey_state: &Option<crate::yubikey::YubiKeyState>, state: &KeyState)
```

**Step 3 — Add touch policy section to render_key_attributes:**
After the three existing slot lines (Signature, Encryption, Authentication) and before the final Paragraph render, add:

```rust
// Touch policies from YubiKeyState
if let Some(ref yk) = yubikey_state {
    if let Some(ref tp) = yk.touch_policies {
        lines.push(Line::from(""));
        lines.push(Line::from(vec![Span::styled(
            "Touch Policies:",
            Style::default().fg(Color::Yellow).add_modifier(Modifier::BOLD),
        )]));
        lines.push(Line::from(vec![
            Span::styled("  Signature:      ", Style::default().fg(Color::Yellow)),
            Span::raw(format!("{}", tp.signature)),
        ]));
        lines.push(Line::from(vec![
            Span::styled("  Encryption:     ", Style::default().fg(Color::Yellow)),
            Span::raw(format!("{}", tp.encryption)),
        ]));
        lines.push(Line::from(vec![
            Span::styled("  Authentication: ", Style::default().fg(Color::Yellow)),
            Span::raw(format!("{}", tp.authentication)),
        ]));
        lines.push(Line::from(vec![
            Span::styled("  Attestation:    ", Style::default().fg(Color::Yellow)),
            Span::raw(format!("{}", tp.attestation)),
        ]));
    }
}
```

This reuses the identical rendering pattern already used in `render_main` (lines ~247-268 of keys.rs). Do not import anything new — `TouchPolicies` Display impl is already in scope via the existing use statements.

<task type="auto">
  <name>Task 2: Call refresh_ssh_status on every Screen::SshWizard entry</name>
  <files>src/app.rs</files>
  <action>
The root cause: `SshState::default()` hard-codes all status booleans to `false`. `refresh_ssh_status()` is only called when actions execute — never on screen entry.

**Two navigation arms to fix:**

**Fix 1 — Char('5') keybind (line ~802):**
Change:
```rust
KeyCode::Char('5') => self.current_screen = Screen::SshWizard,
```
to:
```rust
KeyCode::Char('5') => {
    self.current_screen = Screen::SshWizard;
    self.refresh_ssh_status()?;
}
```

**Fix 2 — Dashboard menu option 3 (line ~760):**
The menu match arm sets `target = Screen::SshWizard` then later runs `self.current_screen = target`. After that assignment, add a check:
```rust
self.current_screen = target;
if target == Screen::SshWizard {
    self.refresh_ssh_status()?;
}
```

Both `refresh_ssh_status` calls return `Result<()>` — propagate with `?` in both places (the enclosing fn already returns `Result<()>`).

Do not reset `self.ssh_state` to default on navigation — that would lose the wizard's sub-screen state. Only refresh the status booleans.
