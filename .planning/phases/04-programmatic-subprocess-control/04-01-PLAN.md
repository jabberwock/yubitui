    Create src/yubikey/gpg_status.rs with:

    1. `GpgStatus` enum with variants:
       - `NeedPassphrase`
       - `GetHidden { prompt: String }` (gpg asks for hidden input like PIN)
       - `GetLine { prompt: String }` (gpg asks for visible input)
       - `GotIt`
       - `Error { operation: String, code: u64 }`
       - `CardCtrl(u32)` (card events: 1=inserted, 3=removed, 4=card-check)
       - `KeyCreated { key_type: String, fingerprint: String }`
       - `ScOpSuccess` (smartcard operation success)
       - `ScOpFailure(u32)` (smartcard operation failure with reason)
       - `PinentryLaunched` (we suppress this since we handle PIN in-TUI)
       - `KeyConsidered { fingerprint: String }`
       - `Progress { what: String, cur: u32, total: u32 }`
       - `Unknown(String)`

    2. `pub fn parse_status_line(line: &str) -> GpgStatus`:
       - Strip `[GNUPG:] ` prefix (return Unknown if missing)
       - Split remainder on whitespace, match first token to variant
       - Parse numeric args with `.parse::<u64>()` / `.parse::<u32>()`, default 0 on failure
       - For ERROR: second token is operation, third is code
       - For KEY_CREATED: second token is key_type (P/B/S), third is fingerprint
       - For GET_HIDDEN/GET_LINE: second token is prompt name
       - For CARDCTRL/SC_OP_FAILURE: second token is numeric code
       - For PROGRESS: tokens are what, cur, total

    3. `pub fn status_to_message(status: &GpgStatus) -> String`:
       - NeedPassphrase -> "PIN required..."
       - GetHidden { prompt } -> match prompt: "passphrase.pin" -> "Enter User PIN", "passphrase.admin_pin" -> "Enter Admin PIN", _ -> "Enter passphrase"
       - GotIt -> "PIN accepted"
       - Error { operation, code } -> match (operation, code):
         - ("change_passwd", 67108949) -> "Incorrect PIN (check remaining attempts)"
         - ("change_passwd", _) -> "PIN change failed"
         - (_, 67108949) -> "Authentication failed"
         - _ -> format!("Operation failed (error {})", code)
       - CardCtrl(3) -> "Card removed -- reinsert and retry"
       - CardCtrl(1) -> "Card detected"
       - ScOpSuccess -> "Operation completed successfully"
       - ScOpFailure(2) -> "Wrong PIN"
       - ScOpFailure(_) -> "Smartcard operation failed"
       - KeyCreated { .. } -> "Key generated successfully"
       - PinentryLaunched -> "" (empty, ignored)
       - Progress { what, cur, total } -> format!("{}: {}/{}", what, cur, total)
       - Unknown(_) -> "" (empty, ignored)

    4. Add `pub mod gpg_status;` to src/yubikey/mod.rs

    5. Write tests as described in <behavior> above, using string fixture inputs (no hardware needed). Per D-08: translate common tokens to human-readable messages. Follow the Phase 3 parser testing pattern (parse function separated from command invocation, tested with string fixtures).

         - `KeyCode::Char(c)` if c.is_ascii_graphic() -> append to active field value, return `PinInputAction::Continue`
