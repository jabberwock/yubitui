<remaining_work>

- **Bug**: `factory_reset_openpgp` — TERMINATE DF (00 E6 00 00) returns SW 6982 (Security Status Not Satisfied) even though admin PIN is 0/3 retries (fully blocked).

  **Root cause hypothesis**: OpenPGP spec says TERMINATE DF security condition is "Admin-PIN blocked OR Admin-PIN verified" — both are evaluated per-session, not from the persistent retry counter alone. The card doesn't know admin is blocked until a VERIFY attempt is made in the current session.

  **Fix**: Before TERMINATE DF, issue VERIFY admin with a dummy/wrong 8-byte PIN:
  ```
  00 20 00 83 08 FF FF FF FF FF FF FF FF
  ```
  Card will return 6983 (Authentication method blocked). This establishes the per-session "admin is blocked" security state. Then TERMINATE DF should return 9000.

- Commit all 8 modified files once factory reset is working
</remaining_work>
