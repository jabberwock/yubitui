  <name>Task 1: Add des crate + create PIV delete model module</name>
  <files>Cargo.toml, src/model/piv_delete.rs, src/model/mod.rs</files>
  <action>
1. Add `des` crate to Cargo.toml dependencies:
   ```toml
   des = "0.8"
   ```
   Also add `cipher` (the BlockEncrypt trait provider):
   ```toml
   cipher = "0.4"
   ```
   NOTE: Use des 0.8 (stable release) rather than 0.9.0-rc.3 (pre-release). The 0.8 series provides TdesEde3 and BlockEncrypt via the cipher 0.4 traits. If 0.8 does not have TdesEde3, fall back to 0.9.0-rc.3 with `des = { version = "=0.9.0-rc.3" }`.

2. Create `src/model/piv_delete.rs` with:

   a. `PivSlot` enum (Authentication, Signature, KeyManagement, CardAuth) with methods:
      - `object_id_bytes(&self) -> &'static [u8]` — returns 3-byte object tag (0x5F 0xC1 0x05/0x0A/0x0B/0x01)
      - `slot_id(&self) -> u8` — returns 0x9A/0x9C/0x9D/0x9E
      - `display_name(&self) -> &'static str` — "Authentication (9a)", etc.
      - `from_slot_str(s: &str) -> Option<Self>` — parses "9a"/"9c"/"9d"/"9e"

   b. `PIV_DEFAULT_MGMT_KEY_3DES: &[u8; 24]` — the well-known default `01 02 03 04 05 06 07 08` repeated 3x.

   c. `pub fn authenticate_piv_mgmt_key_3des(card: &pcsc::Card, key: &[u8; 24]) -> Result<()>`:
      - Step 1: Send GENERAL AUTHENTICATE challenge request: `[0x00, 0x87, 0x03, 0x9B, 0x04, 0x7C, 0x02, 0x81, 0x00]`
      - Check SW 0x9000, extract 8-byte challenge from response at bytes [4..12] (layout: `7C 0A 81 08 [8 bytes]`)
      - Step 2: Encrypt challenge with 3DES-EDE (Triple DES):
        ```rust
        use des::TdesEde3;
        use cipher::{BlockEncrypt, KeyInit};
        use cipher::generic_array::GenericArray;
        let cipher = TdesEde3::new(GenericArray::from_slice(key));
        let mut block = GenericArray::clone_from_slice(&challenge[..8]);
        cipher.encrypt_block(&mut block);
        ```
      - Build step 2 APDU: `[0x00, 0x87, 0x03, 0x9B, 0x0C, 0x7C, 0x0A, 0x82, 0x08, ...8 encrypted bytes]`
      - Transmit and check SW 0x9000. On failure: "Management key authentication failed (wrong key?)"

   d. `pub fn delete_piv_certificate(card: &pcsc::Card, slot: &PivSlot) -> Result<()>`:
      - Build TLV data: `[0x5C, obj_id_len, ...obj_id_bytes, 0x53, 0x00]`
      - PUT DATA: `[0x00, 0xDB, 0x3F, 0xFF, data_len, ...data]`
      - Check SW 0x9000

   e. `pub fn delete_piv_key(card: &pcsc::Card, slot: &PivSlot, firmware: &crate::model::Version) -> Result<()>`:
      - Version gate: if `major < 5 || (major == 5 && minor < 7)`, bail with clear message
      - MOVE KEY: `[0x00, 0xF6, 0xFF, slot.slot_id()]`
      - Check SW 0x9000

   f. `pub fn delete_piv_slot(slot: &PivSlot, mgmt_key: &[u8; 24], firmware: &crate::model::Version) -> Result<String>`:
      - High-level function: kill_scdaemon + 50ms sleep, connect to PIV card, SELECT PIV, authenticate management key, delete certificate, then attempt key delete if firmware >= 5.7.
      - Returns a status message: "Certificate deleted" or "Certificate and key deleted" or "Certificate deleted (key deletion requires firmware 5.7+, yours is X.Y.Z)"
      - This is the function the TUI calls.

3. Add `pub mod piv_delete;` to `src/model/mod.rs`.

IMPORTANT: The management key auth step 2 APDU Lc byte is 0x0C (12 bytes of data: 7C 0A 82 08 + 8 encrypted bytes). Double-check the TLV wrapping.

IMPORTANT: Use the PIV connect pattern from `get_piv_state()` — kill_scdaemon, 50ms sleep, establish context, connect exclusive, SELECT PIV. Do NOT use `connect_to_openpgp_card()` — that selects the OpenPGP AID.

2. Add 'D' keybinding (`action: "delete_slot"`, `description: "D Delete"`, `show: true`) to the PIV screen's key_bindings.

3. In `on_action("delete_slot")`:
   - Get the selected slot string ("9a"/"9c"/"9d"/"9e") from the selected_slot index
   - Check if the slot is occupied (look in `yubikey_state.piv.slots` for a matching entry)
   - If not occupied: push PopupScreen("Empty Slot", "No certificate or key to delete in this slot.")
   - If occupied: push `ModalScreen::new(Box::new(MgmtKeyThenDeleteScreen::new(piv_slot, firmware_version)))`

4. Create `MgmtKeyThenDeleteScreen` struct in piv.rs:
   - Collects management key as hex string input (48 hex chars = 24 bytes)
   - Shows: "Enter PIV Management Key (hex)\nDefault key: press Enter to use default"
   - Pre-fills with default key display for convenience
   - On Enter with empty input: use `PIV_DEFAULT_MGMT_KEY_3DES`
   - On Enter with input: parse hex string to `[u8; 24]`, show error if invalid
   - On successful key parse: push `DeletePivConfirmScreen::new(slot, key, firmware)`
   - On Esc: pop_screen_deferred

5. Create `DeletePivConfirmScreen` struct wrapping ConfirmScreen:
   - Body text: "Permanently delete contents of PIV slot {slot.display_name()}?\n\n{firmware_message}\n\nThis cannot be undone."
   - `firmware_message`: if >= 5.7.0 "Both certificate and key will be deleted." else "Certificate will be deleted. Key material cannot be removed on firmware {version} (requires 5.7+)."
   - on_action("confirm"): call `crate::model::piv_delete::delete_piv_slot(&slot, &key, &firmware)`
     - On Ok(msg): pop, push PopupScreen("Success", msg)
     - On Err(e): pop, push PopupScreen("Error", format!("Delete failed: {}", e))
   - on_action("cancel"): pop_screen_deferred

6. Use `on_event` with KeyEvent downcast for character-level hex input in MgmtKeyThenDeleteScreen (same pattern as AddAccountScreen in oath.rs and PinAuthScreen in fido2.rs).

IMPORTANT: The firmware Version struct is available from `yubikey_state.as_ref().map(|yk| &yk.info.version)`. Clone it for the delete screen.
