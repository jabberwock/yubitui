<objective>
Wire refresh actions on OathScreen and Fido2Screen (both currently no-op stubs), and fix PIV post-delete to use the already-fetched fresh state instead of discarding it.

Purpose: OATH and FIDO2 screens always receive None from YubiKeyState because detection.rs intentionally skips them (expensive). Without a refresh handler, these screens can never show card data after initial navigation. The PIV fix ensures users see updated slot state after a delete without manually pressing R.

Output: Three files modified with working refresh/post-delete state handlers.
</objective>

Replace with code that constructs a minimal YubiKeyState with the fresh PIV data. The approach:

// Re-detect full YubiKey state so PivScreen shows updated slot data
let fresh_yk = crate::model::YubiKeyState::detect_all()

Remove the `let _ = fresh_piv_state;` line and the `let fresh_piv_state = ...` line since we now use detect_all() instead.
