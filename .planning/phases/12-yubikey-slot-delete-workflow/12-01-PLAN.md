<task type="auto">
  <name>Task 1: Create OpenPGP slot delete model module</name>
  <files>src/model/openpgp_delete.rs, src/model/mod.rs</files>
  <action>
Create `src/model/openpgp_delete.rs` with the following:

1. `OpenPgpKeySlot` enum (Sig, Enc, Aut) with `algorithm_attr_tag()` method returning DO tag bytes (0xC1, 0xC2, 0xC3) and a `display_name()` method returning "Signature", "Encryption", "Authentication".

2. Constants `RSA4096_ATTRS` and `RSA2048_ATTRS` as `&[u8; 6]`:
   - RSA4096: `[0x01, 0x10, 0x00, 0x00, 0x11, 0x00]`
   - RSA2048: `[0x01, 0x08, 0x00, 0x00, 0x11, 0x00]`

3. `pub fn delete_openpgp_key(slot: OpenPgpKeySlot, admin_pin: &str) -> Result<()>`:
   - Call `super::card::kill_scdaemon()` + 50ms sleep
   - Call `super::card::connect_to_openpgp_card()` to get card handle (this already SELECTs OpenPGP AID — Pitfall 6)
   - VERIFY Admin PIN: transmit `[0x00, 0x20, 0x00, 0x83, pin_len, ...pin_bytes]`. Check SW: 0x9000 = success, 0x63Cx = wrong PIN (x retries left), 0x6983 = PIN blocked. Return clear error messages.
   - PUT DATA (0xDA) with tag from `slot.algorithm_attr_tag()` and RSA4096_ATTRS. Abort if SW != 0x9000.
   - PUT DATA (0xDA) again with same tag and RSA2048_ATTRS. Abort if SW != 0x9000.
   - Return Ok(()) on success.

4. Add `pub mod openpgp_delete;` to `src/model/mod.rs`.

Follow existing card.rs patterns: `transmit()` with fixed-size buffer, `apdu_sw()` for status word extraction, `apdu_error_message()` for human-readable errors. The function takes ownership of card connection (connects fresh each time, matching `get_openpgp_state()` pattern).

IMPORTANT: For VERIFY PIN error handling, match on SW high byte 0x63 with the low nibble as retry count. Return error like "Wrong Admin PIN (N retries remaining)" or "Admin PIN is blocked" for 0x6983.
