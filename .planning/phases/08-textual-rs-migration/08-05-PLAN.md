   The current keys.rs has KeyGenWizard with steps: SelectSlot, SelectAlgorithm, SelectSize, ConfirmOverwrite, Generating, Success, Failure.

   Convert to a `KeyGenWizardScreen` implementing Widget:
   - Internal reactive state tracks the current step

   - Step transitions update reactive state (not push/pop per step — that would be too many screen levels)
   - Pushed via `ctx.push_screen_deferred(Box::new(KeyGenWizardScreen::new(...)))` from on_action("generate_key")
   - Esc pops back to Keys screen (cancels wizard)
   - The "Generating" step shows progress widget
   - Success/Failure are terminal states with "Done" button that pops

   **ImportKeyFlow**:
   Similar pattern — pushed screen with step-by-step flow.

   **DeleteKeyConfirmation**:
   Use PopupScreen (from Plan 04) with $error styling per UI-SPEC destructive pattern:
   - Header: "Delete Slot -- This will erase the key and certificate in this slot."
   - Confirm button: "Delete Slot" ($error styled)
   - Cancel button: auto-focused default

   - Key slot display (algorithm, size, creation date, fingerprint)
   - No-keys state: "No keys configured" heading + "Generate or import keys using the buttons below." body (per UI-SPEC copywriting)
