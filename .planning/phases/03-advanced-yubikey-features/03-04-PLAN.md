  <action>
    Replace the entire `.github/workflows/rust.yml` with a 3-OS matrix strategy. The new workflow:

    ```yaml
    name: CI

    on:
      push:
        branches: [ "main" ]
      pull_request:
        branches: [ "main" ]

    env:
      CARGO_TERM_COLOR: always

    jobs:
      build-and-test:
        strategy:
          fail-fast: false
          matrix:
            include:
              - os: ubuntu-latest
                name: Linux
              - os: macos-latest
                name: macOS
              - os: windows-latest
                name: Windows
        runs-on: ${{ matrix.os }}
        name: ${{ matrix.name }}

        steps:
          - uses: actions/checkout@v4

          - name: Install system dependencies (Linux only)
            if: runner.os == 'Linux'
            run: sudo apt-get update && sudo apt-get install -y libpcsclite-dev

          - name: Build
            run: cargo build --verbose

          - name: Run tests
            run: cargo test --verbose

          - name: Clippy
            run: cargo clippy -- -D warnings
    ```

    Key differences from existing:
    - Job renamed from `build` to `build-and-test`
    - `strategy.matrix.include` with 3 OS entries
    - `fail-fast: false` so all OS results are visible even if one fails
    - Linux-only conditional for libpcsclite-dev install
    - macOS and Windows need NO extra system deps (PCSC.framework and winscard.dll are native)
    - Added `cargo clippy -- -D warnings` step
    - Tests run without `--features device-tests` (no YubiKey on CI runners)
  </action>

<task type="auto">
  <name>Task 2: Create release workflow for binary distribution</name>
  <files>.github/workflows/release.yml</files>
  <read_first>.github/workflows/rust.yml, Cargo.toml</read_first>
  <action>
    Create `.github/workflows/release.yml` with the following content:

    ```yaml
    name: Release

    on:
      push:
        tags:
          - "v*"

    permissions:
      contents: write

    env:
      CARGO_TERM_COLOR: always

    jobs:
      build-release:
        strategy:
          fail-fast: false
          matrix:
            include:
              - os: ubuntu-latest
                target: x86_64-unknown-linux-gnu
                artifact_name: yubitui-linux-amd64
                binary_name: yubitui
              - os: macos-latest
                target: x86_64-apple-darwin
                artifact_name: yubitui-macos-amd64
                binary_name: yubitui
              - os: windows-latest
                target: x86_64-pc-windows-msvc
                artifact_name: yubitui-windows-amd64.exe
                binary_name: yubitui.exe
        runs-on: ${{ matrix.os }}
        name: Build ${{ matrix.os }}

        steps:
          - uses: actions/checkout@v4

          - name: Install system dependencies (Linux only)
            if: runner.os == 'Linux'
            run: sudo apt-get update && sudo apt-get install -y libpcsclite-dev

          - name: Build release binary
            run: cargo build --release --verbose

          - name: Run tests
            run: cargo test --verbose

          - name: Rename binary
            shell: bash
            run: |
              cp target/release/${{ matrix.binary_name }} ${{ matrix.artifact_name }}

          - name: Upload artifact
            uses: actions/upload-artifact@v4
            with:
              name: ${{ matrix.artifact_name }}
              path: ${{ matrix.artifact_name }}

      create-release:
        needs: build-release
        runs-on: ubuntu-latest
        name: Create Release

        steps:
          - uses: actions/checkout@v4

          - name: Download all artifacts
            uses: actions/download-artifact@v4
            with:
              path: artifacts

          - name: Create GitHub Release
            uses: softprops/action-gh-release@v2
            with:
              generate_release_notes: true
              files: |
                artifacts/yubitui-linux-amd64/yubitui-linux-amd64
                artifacts/yubitui-macos-amd64/yubitui-macos-amd64
                artifacts/yubitui-windows-amd64.exe/yubitui-windows-amd64.exe
    ```

    Key design decisions:
    - Triggers on `v*` tag push (e.g., `v0.1.0`)
    - `permissions: contents: write` needed for creating releases
    - Each OS builds independently, then a final job creates the GitHub Release
    - Artifact names include OS and architecture to avoid collisions (per research pitfall 5)
    - Windows binary has `.exe` extension (per research pitfall 6)
    - Uses `softprops/action-gh-release@v2` for release creation with auto-generated notes
    - Tests also run during release build to ensure release binaries are from tested code
  </action>
