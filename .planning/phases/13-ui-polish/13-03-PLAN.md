  <action>
Modify OathScreen::compose() to match PIN Management visual standard. Add `Button`, `DataTable`, `ColumnDef`, `ProgressBar` to the `use textual_rs::` import line.

1. **Credential list — replace format-string Labels with DataTable:**
   When credentials exist, replace the manual header/separator/row Labels (lines ~199-251) with a DataTable:
   ```rust
   let columns = vec![
       ColumnDef::new("", 2),           // Selection cursor
       ColumnDef::new("Name", 30),
       ColumnDef::new("Code", 14),
       ColumnDef::new("Type", 8),
   ];
   ```

   Build rows from `state.credentials`:
   - cursor: ">" for selected index, " " otherwise
   - name: `cred.issuer.as_deref().unwrap_or(&cred.name)`, truncated to 30 chars
   - code: for TOTP, the code or "------"; for HOTP with no code, "[Enter]"
   - type: "[TOTP]" or "[HOTP]"

   Use `DataTable::new(columns, rows)`.

2. **TOTP countdown — replace ASCII bar with ProgressBar:**
   Replace the manual `"="` repeat / `" "` repeat bar (lines ~256-265) with:
   ```rust
   let now_secs = chrono::Utc::now().timestamp();
   let secs_remaining = 30 - (now_secs % 30);
   let progress = secs_remaining as f64 / 30.0;
   widgets.push(Box::new(Label::new(format!(
       "TOTP refreshes in {}s", secs_remaining
   ))));
   widgets.push(Box::new(ProgressBar::new(progress)));
   ```

   Note: ProgressBar::new takes a f64 from 0.0 to 1.0. Check the actual textual_rs ProgressBar API — if it takes a different signature, adapt accordingly. The key point is replacing ASCII art with the real widget.

3. **Action Buttons:**
   After the countdown/data section and a spacer, add Buttons for the primary actions:
   ```rust
   widgets.push(Box::new(Label::new("")));
   widgets.push(Box::new(Button::new("Add Account (A)")));
   widgets.push(Box::new(Button::new("Delete Account (D)")));
   widgets.push(Box::new(Button::new("Refresh (R)")));
   ```

4. **Empty state — add action button:**
   In the empty credentials branch, after "No OATH credentials stored" Label, add:
   ```rust

   ```

5. **No YubiKey state — add Refresh button:**
   ```rust
   widgets.push(Box::new(Button::new("Refresh (R)")));
   ```

6. **Layout enforcement:**
   Header -> credential DataTable (or empty/password state) -> countdown section -> spacer -> action Buttons -> Footer.
