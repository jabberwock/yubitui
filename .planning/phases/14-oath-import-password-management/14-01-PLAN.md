Replace `ImportUriScreen` struct and its Widget impl with a two-step version.

New struct definition:
```rust
#[derive(Clone, Copy, PartialEq)]
enum ImportUriStep { Paste, Confirm }

pub struct ImportUriScreen {
    input: RefCell<String>,
    step: Cell<ImportUriStep>,
    parsed: RefCell<Option<ParsedOtpAuth>>,
    error: RefCell<Option<String>>,
    own_id: Cell<Option<textual_rs::WidgetId>>,
}
```

`on_event` logic:
- In `Paste` step: Char(c) appends to input, Backspace pops, Enter calls `try_parse()` (see below), Esc pops screen
- In `Confirm` step: Enter calls `commit()`, Esc sets step back to Paste and clears error (keeps input for re-editing)

`try_parse()` method:
- Calls `parse_otpauth_uri(&input)` — on Ok sets `self.parsed` and `self.step = Confirm`; on Err sets error message, stays in Paste step

`commit()` method:
- Reads `self.parsed`, calls `put_credential(name, secret_b32, oath_type, algorithm, digits)`
- On Ok: `ctx.pop_screen_deferred()` + push `PopupScreen::new("Imported", format!("'{}' added.", name))`
- On Err: sets error, returns to Paste step

`compose()` for Paste step (same as current ImportUriScreen compose):
```
Header("Import OATH URI")
Label("")
Label("Paste an otpauth:// URI and press Enter to preview.")
Label("")
Label("> {input}_")
[error if any]
Label("")
Footer
```

`compose()` for Confirm step:
```
Header("Import OATH URI")
Label("")
Label("Review before adding:")
Label("")
Label(format!("  Issuer:    {}", issuer.as_deref().unwrap_or("(none)")))
Label(format!("  Account:   {}", account))
Label(format!("  Secret:    {}", "*".repeat(secret_b32.len().min(12))))
Label(format!("  Algorithm: {}", algorithm))
Label(format!("  Type:      {}", oath_type))
Label(format!("  Digits:    {}", digits))
Label("")
Label("Press Enter to add, Esc to go back and edit.")
[error if any]
Label("")
Footer
```

Key bindings: same `IMPORT_URI_BINDINGS` static (Esc=cancel, Enter=import).

Note: In Confirm step, Esc should return to Paste step (not pop the whole screen).
Handle this in `on_event` rather than `on_action` since the behavior differs by step.
The `on_action` "cancel" handler should pop the screen (called from Footer Esc when already in Paste).
