    ```rust
    pub const OATH_AID: &[u8] = &[0xA0, 0x00, 0x00, 0x05, 0x27, 0x21, 0x01, 0x01];
    pub const SELECT_OATH: &[u8] = &[0x00, 0xA4, 0x04, 0x00, 0x08, 0xA0, 0x00, 0x00, 0x05, 0x27, 0x21, 0x01, 0x01];
    // LIST: CLA=00 INS=A1 P1=00 P2=00
    pub const LIST_CREDENTIALS: &[u8] = &[0x00, 0xA1, 0x00, 0x00];
    // CALCULATE ALL: CLA=00 INS=A4 P1=00 P2=01 (P2=01 means truncate)
    // Challenge data appended dynamically
    pub const CALCULATE_ALL_PREFIX: &[u8] = &[0x00, 0xA4, 0x00, 0x01];
    // PUT: CLA=00 INS=01 P1=00 P2=00 (data TLV appended)
    pub const PUT_CREDENTIAL_PREFIX: &[u8] = &[0x00, 0x01, 0x00, 0x00];
    // DELETE: CLA=00 INS=02 P1=00 P2=00 (name TLV appended)
    pub const DELETE_CREDENTIAL_PREFIX: &[u8] = &[0x00, 0x02, 0x00, 0x00];
    ```

    **OATH TLV tag constants:**
    ```rust
    const TAG_NAME: u8 = 0x71;
    const TAG_KEY: u8 = 0x73;
    const TAG_CHALLENGE: u8 = 0x74;
    const TAG_RESPONSE: u8 = 0x75;
    const TAG_TRUNCATED: u8 = 0x76;
    const TAG_PROPERTY: u8 = 0x78;
    const TAG_IMF: u8 = 0x7A;  // Initial moving factor (HOTP counter)
    ```

    **Helper functions:**
    - `pub fn calculate_timestep(unix_secs: i64) -> [u8; 8]` — returns `(unix_secs / 30) as u64` in big-endian bytes
    - `fn parse_tlv(data: &[u8]) -> Vec<(u8, Vec<u8>)>` — parse OATH TLV stream: each entry is tag(1 byte) + length(1 byte, or 0x81+1 byte if >127) + value(length bytes)
    - `fn parse_list_response(data: &[u8]) -> Vec<OathCredential>` — parse LIST response TLVs (TAG_NAME entries, byte 0 of value encodes type|algorithm)
    - `fn parse_calculate_all_response(data: &[u8], credentials: &mut Vec<OathCredential>)` — parse CALCULATE ALL response, matching name TLVs to truncated code TLVs, extracting dynamic code from 4-byte truncated value as `code % 10^digits` zero-padded
    - `fn build_calculate_all_apdu(timestep: [u8; 8]) -> Vec<u8>` — CALCULATE ALL prefix + Lc + TAG_CHALLENGE + len(8) + timestep bytes
    - `fn build_put_apdu(name: &str, secret: &[u8], oath_type: OathType, algorithm: OathAlgorithm, digits: u8) -> Vec<u8>` — PUT prefix + Lc + TAG_NAME + name + TAG_KEY + (type|algo byte + digits byte + secret)
    - `fn build_delete_apdu(name: &str) -> Vec<u8>` — DELETE prefix + Lc + TAG_NAME + name bytes

    **Public card functions (follow piv.rs pattern):**
    - `pub fn get_oath_state() -> Result<OathState>` — kill_scdaemon, sleep 50ms, pcsc connect, SELECT OATH (check SW: 0x6982 = password_required, 0x9000 = ok), LIST, CALCULATE ALL with current timestep, return OathState
    - `pub fn calculate_all(credentials: &mut Vec<OathCredential>) -> Result<()>` — same connection flow but only CALCULATE ALL (for refresh)
    - `pub fn put_credential(name: &str, secret_b32: &str, oath_type: OathType, algorithm: OathAlgorithm, digits: u8) -> Result<()>` — Base32-decode secret (manual: A-Z2-7 alphabet, no external crate), build PUT APDU, transmit, check SW 0x9000
    - `pub fn delete_credential(name: &str) -> Result<()>` — build DELETE APDU, transmit, check SW 0x9000

    3. Use `base64` crate's general purpose alphabet for Base32 decoding — actually, implement a simple Base32 decode inline (RFC 4648 A-Z2-7 alphabet, ~20 lines) to avoid adding another dependency. The function signature: `fn base32_decode(input: &str) -> Result<Vec<u8>>`.
