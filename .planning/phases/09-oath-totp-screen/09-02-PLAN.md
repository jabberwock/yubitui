    **Widget impl — compose():**
    - Header("OATH Credentials")
    - If oath_state is None or password_required: show appropriate Label message
    - If credentials empty: Label("No OATH credentials stored. Press 'a' to add one.")
    - For each credential, render a Label row:
      - Format: `"  {marker} {issuer_or_name:<30} {code:>8}  {type_badge}"`
      - `marker`: `">"` for selected_index, `" "` otherwise
      - `issuer_or_name`: credential.issuer or credential.name
      - `code`: credential.code.as_deref().unwrap_or("[press Enter]") for HOTP without code, or the 6/8 digit code for TOTP
      - `type_badge`: `"[TOTP]"` or `"[HOTP]"`
    - Countdown bar Label: `"  TOTP refreshes in {secs_remaining}s  {bar}"` where:
      - `secs_remaining = 30 - (chrono::Utc::now().timestamp() % 30)`
      - `bar` = a simple text bar like `"[========        ]"` proportional to secs_remaining/30 (use `"="` chars for filled, `" "` for empty, 20 chars wide)
    - Footer with OATH_BINDINGS

    **Widget impl — on_action():**
    - `"back"` -> `ctx.pop_screen()`
    - `"up"` -> decrement selected_index (clamp to 0)
    - `"down"` -> increment selected_index (clamp to credentials.len()-1)
    - `"generate_hotp"` -> if selected credential is HOTP, this is a placeholder (actual APDU call deferred — just show a popup "HOTP generation requires card access" for now; full implementation in this plan uses CALCULATE with counter)
    - `"add_account"` -> placeholder `ctx.push_screen_deferred(...)` (wired in Plan 03)
    - `"delete_account"` -> placeholder (wired in Plan 03)
    - `"refresh"` -> placeholder (would re-CALCULATE ALL; no-op in mock mode)

    **IMPORTANT design decision from NOTES.md gray areas:**
    - **Live refresh**: Local clock tick every second for the countdown bar. Only hit the card at the 30s boundary (when `secs_remaining` wraps from 1 back to 30). In this plan, the countdown is computed on each render from `chrono::Utc::now()` — no timer thread needed since textual-rs re-renders on any key event.
    - **Credential row density**: Full row (name + code + type badge + countdown bar at bottom).
    - **HOTP behavior**: Require explicit Enter press to generate (show `"[press Enter]"` placeholder).
