## Overview

Elevated yubitui from feature-complete to goal-oriented. Added outcome-driven provisioning wizards, OATH credential import via URI paste with two-step preview, full OATH application password lifecycle management, PIV management key change with factory-default detection, and dashboard navigation affordances to help new users discover all screens.

## Phases

### Phase 14: OATH Import & Password Management

**Goal**: Users can import OATH accounts via URI paste and fully manage the OATH application password
**Depends on**: Phase 13 (OATH screen already exists; extends it)
**Requirements**: OATH-07, OATH-08, OATH-09, OATH-10
**Plans**: 3 plans (14-01, 14-02, 14-03)

Plans:

- [x] 14-01: Two-step otpauth:// URI import with preview (OATH-07)
- [x] 14-02: OATH password set/change (OATH-08, OATH-09)
- [x] 14-03: OATH password remove (OATH-10)

**Success Criteria — All Met:**
1. ✅ User pastes an otpauth:// URI and sees issuer, account, secret (masked), and algorithm pre-filled before confirming
2. ✅ User can set an OATH application password when none exists; subsequent operations prompt via SW 0x6982
3. ✅ User can change the OATH password after authenticating with the current password
4. ✅ User can remove the OATH password after authenticating, returning applet to unprotected mode

**Key implementation details:**
- `ImportUriStep` enum (Paste → Confirm) with `ParsedOtpAuth` struct for type-safe preview data
- Algorithm parsed from URI (`sha256`, `sha512`, default SHA1)
- Secret displayed as first 4 chars + asterisks for security review without full exposure
- `parse_otpauth_uri()` returns `Result<ParsedOtpAuth, String>` (was tuple)
- OATH password auth via HMAC-SHA1 challenge-response (INS=0xA4/0x89)

## Milestone Summary

**Key Decisions:**
- Two-step URI import: parse first, preview second — reduces fat-finger OATH account imports
- Secret masking (first 4 + asterisks): shows enough for user to verify without full secret exposure in TUI
- `ParsedOtpAuth` struct over tuple return: type safety for multi-field OATH data
- PIV management key probe in `get_piv_state()`: no extra round-trip; factory-default warning available at screen load
- Wizard as separate screen (`src/tui/wizard.rs`): clean separation from per-applet screens

**Issues Resolved:**
- All 3 CI platforms (Linux/macOS/Windows) were broken by `to_string_in_format_args` clippy lint in `src/model/piv.rs:225-226` — fixed and merged via PR #1
- GitHub repo ruleset requiring CodeQL for Rust (unsupported) blocked all pushes — ruleset deleted (ID 14485617)
- OATH URI parser return type changed from tuple to struct — all tests updated

**Issues Deferred:**
- Mouse click on Button widgets: known textual-rs 0.3.11 limitation; keyboard workaround works; fix expected in 0.3.12
- OTP slot write (Yubico OTP, static password, HMAC-SHA1): underdocumented HID frame protocol, deferred to v2
- FIDO2 fingerprint management: Bio series only, niche hardware

**Technical Debt Incurred:**
- Phase 15–17 executed without formal PLAN.md files (agents worked directly from ROADMAP success criteria)
- No SUMMARY.md files written for any v1.2 phase
