**Status:** ✅ SHIPPED 2026-03-26

## Overview

First complete release of yubitui — a terminal user interface for YubiKey management, written in Rust. Covers the full journey from initial polish and cross-platform fixes through to complete elimination of all external CLI dependencies (ykman, gpg --card-status) via native PC/SC APDUs.

## Milestone Summary

**Key Decisions:**
- PC/SC exclusive mode with scdaemon kill-before / restart-after — avoids SW 0x6B00 contention
- card.rs is the single PC/SC APDU primitive module — all card reads go through it
- T=0 GET RESPONSE chaining in get_data() handles multi-part card responses transparently
- gpg remains for keyring operations only; all card reads are native APDUs
- --pinentry-mode loopback for non-interactive PIN operations
- 7-step wizard UI for key generation; auto-map subkeys by capability for import
- previous_screen field pattern for modal overlays (help, attestation popup)
- Vec<YubiKeyState> with selected index for multi-key support
- Layout-based centering for popups (Rect::centered() doesn't exist in ratatui 0.29)
- gpgconf --list-dirs homedir as authoritative gnupg path source

**Issues Resolved:**
- YubiKey 5C NFC showing as "YubiKey NEO" — fixed GET_DEVICE_INFO 0x71 outer TLV unwrap
- T=0 GET RESPONSE chaining for fingerprint (0xC5) reads on YubiKey 5.4.x
- SSH agent false negative on Windows (wrong gnupg conf path)
- gpg --card-edit leaking to terminal on PIN operations
- scdaemon contention causing SW 0x6B00 on exclusive card access

**Tech Debt Incurred:**
- cargo fmt diffs in src/app.rs, src/ui/pin.rs, src/ui/widgets/popup.rs, src/utils/config.rs
- dead_code: show_context_menu and menu_selected_index on KeyState (reserved for future context menu)
- Phase 04 VERIFICATION.md missing (functional evidence strong from Phase 05 verifier)
- Char('t') nav arm doesn't clear key_state.message (cosmetic: stale message visible on touch policy screen)
- No 50ms sleep after kill_scdaemon() — may cause Card Busy on slow Linux scdaemon teardown

**Backlog (Not In Scope for v1.0):**
- FIDO2/WebAuthn status display
- Configurable refresh interval
- Export key directly to clipboard
- Reactive ratatui rendering engine (componentization of app.rs God struct)
